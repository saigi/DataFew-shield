What is Datafew Shield?

Execution Safety

LLM content filters control what the model says. Datafew Shield controls what the agent does. When an agent calls read_file("/etc/shadow"), no prompt guard or moderation API can stop it — Shield can.

Model-Independent

Shield operates at the tool-call layer, independent of the LLM provider. The same policies protect GPT-4, Claude, MiniMax, and open-source models. No model change requires policy changes.

Architecture
Input → Sandbox → Policy Engine → Data Classifier → Chain Detector → Vector Scoring → Session Risk
         (L0)      (L0)            (L2)             (L0)            (L1)           (L3)

Deterministic layers (zero false positives by design) execute before probabilistic layers.
DLP: data lineage tracking between L2 and L3 — read sensitive → egress blocked.

Deterministic Layers (L0, L2)

  • Sandbox — session isolation
  • Policy Engine — 20+ rules (rm -rf, reverse shell, exfil)
  • Data Classifier — P0–P4 path + content classification
  • Chain Detector — multi-step attack pattern matching

Probabilistic Layers (L1, L3)

  • Vector Scoring — transformer + LR on 384-dim embeddings
  • Session Risk — 3-dimensional risk (policy, semantic, behavior)
  • P2 Output — post-generation LLM output safety check
  • DLP — data taint tracking + egress detection

Quick Start
git clone https://github.com/saigi/DataFew-shield.git
cd DataFew-shield
npm install && pip install -r requirements.txt

python scripts/embedding_server.py --port 5000 &
node index.js --port 8080

# Test it
curl -X POST http://localhost:8080/inspect \
  -H "Content-Type: application/json" \
  -d '{"type":"tool_call","tool":"terminal","args":"cat /etc/shadow","session_id":"demo"}'

# Run all benchmarks
bash demo.sh
