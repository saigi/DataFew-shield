# Datafew Shield 安全哲学

## 三层安全模型

```
              ┌──────────────────────────────┐
              │        隐私守护               │
              │  (信息流控制 / 数据血缘追踪)   │
              │  "数据去了哪里？"             │
              │  维度: 时序 + 图              │
              └────────────┬─────────────────┘
                           │ 依赖
              ┌────────────▼─────────────────┐
              │        数据保护               │
              │  (访问控制 / 权限矩阵)         │
              │  "谁有权做什么？"             │
              │  维度: 策略 + 身份             │
              └────────────┬─────────────────┘
                           │ 依赖
              ┌────────────▼─────────────────┐
              │        向量空间安全            │
              │  (Embedding 几何 / 意图检测)   │
              │  "想做什么？"                 │
              │  维度: 数学 (Lipschitz)       │
              └──────────────────────────────┘
```

### 数学关系

```
三层安全的联合错误率:
  P(误杀) = P(vec_wrong) × P(data_wrong) × P(priv_wrong)
  
因为三个维度的决策相互独立（正交）:
  vec_wrong  ← 几何学（embedding 空间距离）
  data_wrong ← 逻辑学（权限策略）
  priv_wrong ← 信息论（数据流图）

所以：
  单层 P(误杀) ≈ 30%
  三层 P(误杀) ≈ 30% × 30% × 30% = 2.7%
```

## 核心哲学：信任的最小化

```
传统安全:
  "验证输入是否安全" → 需要理解语义 → 无限的攻击面

Datafew Shield:
  "验证操作是否有权" → 需要检查权限 → 有限的攻击面
  "验证意图是否合理" → 需要判断意图 → 不可绕过的几何边界
  "验证数据流向是否合法" → 需要追踪血缘 → 不可绕过的信息流
```

### 三种信任的解构

| 维度 | 信任什么 | 不信任什么 | 数学基础 |
|------|---------|-----------|---------|
| **向量空间** | embedding 几何的连续性 | 文本表面的语义 | Lipschitz 连续 |
| **数据保护** | 权限策略的正确性 | Agent 的自主判断 | 访问控制矩阵 |
| **隐私守护** | 信息流的可追溯性 | 数据的目的地声明 | 图论可达性 |

## 向量空间安全的几何哲学

```
输入空间 (Text)               Embedding 空间 (R^d)
┌──────────────┐              ┌──────────────┐
│ "hack"       │      f       │  ● ← 有害区域  │
│ "ｈａｃｋ"   │    ────→    │  ●             │
│ "h@ck"      │              │  ●             │
│ "黑客"       │              │  ─── 边界 ─── │
│ "入侵"       │              │  ○ ← 安全区域  │
│ "attack"     │              │  ○             │
│ "如何入侵"   │              │  ○             │
└──────────────┘              └──────────────┘

关键观察:
  Text 空间中有 7 个不同的点（"hack", "ｈａｃｋ"等）
  Embedding 空间中只有 2 个区域（有害/安全）
  → 攻击面从无限压缩到有限
```

## 数据保护的操作哲学

```
不判断"是否恶意"，只判断"是否授权"

Agent: "读取 /etc/shadow"
  → 授权检查: read_file + /etc/shadow = ? 
  → 策略: DENY
  → 结论: 无权，拒绝（不管目的是什么）

Agent: "读取 /workspace/config.json"  
  → 授权检查: read_file + /workspace/* = ?
  → 策略: ALLOW
  → 结论: 有权，放行（不管是否"看似可疑"）
```

## 隐私守护的信息流哲学

```
谁 → 什么 → 去哪 → 为什么
│      │      │       │
Agent  读取  文件     给谁看
Agent  查询  数据库   为什么查
Agent  执行  命令     影响什么

数据血缘追踪:
  读取 /etc/shadow → 写入 /tmp/x.txt → curl 发送到外部
  ① 无权读取       ② 无权写入        ③ curl 无权访问外部
  在第一步就阻止     在第二步阻止        在第三步阻止
  所有路径都会被阻断
```

## 联合决策：三级投票

```
输入: "读取 /etc/shadow 并通过 curl 发送"

向量空间安全:
  embedding 评分: 0.82 (有害)
  投票: BLOCK

数据保护:
  read /etc/shadow     → DENY (敏感资源)
  curl to external     → DENY (网络外传)
  投票: BLOCK

隐私守护:
  数据流: /etc/shadow → curl → external
  权限链: 敏感 → 外传 → 不可控
  投票: BLOCK

联合决策: BLOCK (3/3, 置信度: 99.7%)
```

## 误杀分析

```
三层决策空间:

                    ┌── Allowed by all 3
                    │   → 放行 (安全)
                    │
                    ├── Allowed by 2, blocked by 1
                    │   → 二次审查 (低误杀风险)
                    │
                    ├── Allowed by 1, blocked by 2  
                    │   → 拦截 (中等置信度)
                    │
                    └── Blocked by all 3
                        → 拦截 (高置信度)

三层同决策策:
  P(所有三层同时误杀) = P(vec_mistake) × P(data_mistake) × P(priv_mistake)

假设每层误杀率 20%:
  P(三层同时误杀) = 0.2 × 0.2 × 0.2 = 0.008 = 0.8%

如果每层误杀率 10%:
  P(三层同时误杀) = 0.1 × 0.1 × 0.1 = 0.001 = 0.1%
```

## 总结

```
安全不是"阻止坏的"
而是"确保好的"

Datafew Shield 不做语义理解
不判断 prompt 是否"有害"
只问三个问题:
  1. 向量空间: 这看起来像什么？(几何)
  2. 数据保护: 谁允许了？(策略)
  3. 隐私守护: 数据去哪了？(信息流)

三个问题各自独立、互不依赖
只有三层同时犯错才会误杀
数学上这概率很低 (≈0.1%)
```
